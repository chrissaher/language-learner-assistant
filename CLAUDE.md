# Multi-Language Learning Platform — Platform Overview

## What This Is

This is a **collection of personalized language learning assistants**, each living in its own subfolder. Each assistant is auto-generated by the `create-assistant` skill, which walks you through a guided Q&A and creates a fully configured folder with:
- A personalized `CLAUDE.md` with your learner profile and language-adapted skill specs
- Four Claude Code skills (sentence builder, grammar explainer, etc.)
- A shared stylesheet and starter HTML pages
- A folder structure ready to receive generated content

Think of it as a **framework for creating customized language learning tools**. Each subfolder is self-contained and portable — you can clone it out and it works standalone.

## How to Use This Platform

### To Create a New Language Assistant
1. Open Claude Code in this root folder (`/Users/csaravia/Documents/personal/language-learner-assistant/`)
2. Type `create assistant` in the chat
3. Answer 10 quick questions about your learning profile, goals, textbook, etc.
4. A new subfolder will be created with everything configured for your language

### To Use an Existing Assistant
1. Open Claude Code in the subfolder (e.g., `french-learning-assistant/`)
2. Use the four skills directly:
   - `sentence builder: [word]` — generate 10 example sentences
   - `render sentences` — convert JSON to styled HTML
   - `grammar: [topic]` — full grammar lesson with exercises
   - `vocab drill: chapter [N], [focus]` — textbook vocabulary with enrichment
3. Generated content (JSON + HTML) accumulates in `data/` — refresh `index.html` with the renderer

### Portable & Standalone
Each subfolder's `CLAUDE.md` contains the complete learner profile and skill specifications. You don't need this platform file to use an assistant — you can clone just the subfolder and it's fully functional.

## Design Principles

### 1. Self-Contained Subfolders
Each assistant is independent:
- All generated content lives in `data/` within the subfolder
- The stylesheet is copied (not shared) so subfolders can be moved or deployed alone
- HTML files link to a local `../../styles.css` (works from `data/*/`)
- No cross-folder dependencies at runtime

### 2. Personalized by Default
The `create-assistant` skill asks 10 questions about:
- Target language and your current level
- Native language (for transfer notes)
- Learning goal and context
- Textbook or curriculum (if applicable)
- Correction style preference

All generated content is adapted to your answers.

### 3. Language Adaptation via Claude's Knowledge
The skill doesn't hardcode per-language rules. Instead, it:
- Synthesizes your target language's grammar properties (tense names, gender system, etc.)
- Writes language-specific skill specifications inline
- Generates exercise types appropriate for that language
- Customizes examples and transfer notes for your native language pair

### 4. Accessible for Humans & Machines
- **index.html** is a human-readable hub
- **CLAUDE.md** in each subfolder is machine-readable context for Claude
- All generated HTML pages are styled and browseable
- Everything is Git-friendly (no binaries, all text)

## The create-assistant Skill

**Location:** `.claude/skills/create-assistant/SKILL.md`

**What it does (in 14 steps):**
1. Introduce the process
2. Ask 6 core profile questions (target language, level, native language, other languages, goal, textbook)
3. Ask 4 follow-up questions (instruction language, correction style, textbook details, special notes)
4. Show a confirmation table and ask for final approval
5. Infer your target language's grammar system, tense names, article system, exercise types, etc.
6. Create the subfolder structure
7. Generate a fully personalized CLAUDE.md
8. Generate 4 adapted skill definition files (sentence-builder, sentence-renderer, grammar-explainer, vocab-drill)
9. Copy the stylesheet
10. Generate a starter index.html
11. Generate a README.md
12. Create a `.claude/settings.json` tool allowlist
13. Update this platform's CLAUDE.md registry table and index.html
14. Show a summary and next steps

**The Q&A flow is conversational:** questions grouped in two rounds, with a confirmation step before file creation. Nothing is generated until you approve.

## Customization & Maintenance

### Updating the Stylesheet
- The `styles.css` at platform root is the canonical version
- Each subfolder has a copy (made at creation time)
- If you update the root `styles.css`, manually copy it to existing subfolders, or run a future `sync-styles` utility

### Adding Assistants Manually
If you want to add an existing assistant without using `create-assistant`:
1. Create a new subfolder (`{language}-learning-assistant/`)
2. Write its personalized CLAUDE.md (copy from the platform CLAUDE.md template, fill in learner profile and language-specific specs)
3. Add the four skill definition files to `.claude/skills/`
4. Add an entry to this file's "Current Assistants" table
5. Add an entry to `index.html`

### Creating Skills Manually in an Existing Assistant
If you open an assistant subfolder and want to create skills individually:
- Read its `CLAUDE.md` for the personalized skill specs
- Ask Claude: "Based on the CLAUDE.md in this folder, help me create a sentence-builder skill"
- Claude will generate the SKILL.md file using the profile context

## Notes for Implementation & Porting

### For New Language Pairs
The `create-assistant` skill adapts automatically to any language by:
- Detecting the target language
- Using Claude's knowledge of that language's grammar system
- Generating language-appropriate skill specifications
- Replacing generic terms (e.g., "Präsens" → correct tense name for target language)
- Customizing transfer notes with learner's actual L1/L2

No per-language hardcoding needed.

### For Non-Latin Scripts (Japanese, Arabic, Mandarin, etc.)
- Subfolder names are ASCII: `japanese-learning-assistant/`, `arabic-learning-assistant/`
- HTML `lang` attributes use correct BCP 47 codes: `ja`, `ar`, `zh-Hans`, etc.
- Slug normalization adapts to romanization or transliteration
- The stylesheet's `system-ui` font stack auto-adapts to any script

## FAQ

**Q: Can I share an assistant subfolder with someone?**
Yes! Each subfolder is standalone. They can clone or download just `french-learning-assistant/`, open it in Claude Code, and it works.

**Q: What if I don't have a textbook?**
The skill asks about your learning focus instead (topics, themes, etc.). The `vocab-drill` skill switches to `vocab drill: topic [name]` mode instead of chapter-based.

**Q: Can I edit the CLAUDE.md in my subfolder?**
Yes! It's just a reference document. Update it whenever your goals or level change — the skills read it as context.

**Q: What if I want to change the stylesheet globally?**
Update the master `styles.css` at the platform root, then copy it to subfolders. (Or leave it as-is if it's working well — the stylesheet is stable.)
